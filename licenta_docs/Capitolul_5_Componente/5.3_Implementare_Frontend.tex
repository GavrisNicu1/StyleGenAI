\section{Dezvoltarea Interfeței Mobile (Frontend)}
\label{sec:impl_frontend}

Interfața StyleGenAI a fost proiectată urmând principiile "Mobile First", punând accent pe rapiditate și claritate vizuală. Implementarea în \textbf{React Native} cu \textbf{TypeScript} a permis crearea unei experiențe native fluide pe ambele platforme majore (iOS/Android).

\subsection{Arhitectura Navigației}
Structura de navigare este coloana vertebrală a aplicației. Am utilizat \textit{Expo Router} pentru a implementa o navigație ierarhică complexă:
\begin{itemize}
    \item \textbf{Stack Navigator}: Pentru fluxul de autentificare (Login -> Register) și detaliile ținutei.
    \item \textbf{Tab Navigator}: Pentru ecranele principale (Home, Explore, History, Profile), accesibile persistent din partea de jos a ecranului.
\end{itemize}

\subsection{Ecranul Principal și Garderoba Virtuală}
Ecranul "Home" servește drept dashboard personal. Implementarea tehnică utilizează componenta `FlatList` pentru a randa eficient grila de haine.

\begin{figure}[h]
    \centering
    \includegraphics[width=0.4\textwidth]{app_home_screen}
    \caption{Ecranul Principal: Vizualizarea grilei de haine încărcate de utilizator.}
    \label{fig:app_home}
\end{figure}

O provocare tehnică a fost gestionarea imaginilor asincrone. Am implementat un sistem de \textit{caching} local pentru a evita re-descărcarea imaginilor la fiecare deschidere a aplicației, reducând consumul de date mobile.

\subsection{Fluxul de Generare a Ținutelor}
Aceasta este funcționalitatea centrală a aplicației. Utilizatorul interacționează cu un formular simplificat, iar rezultatul este afișat sub formă de carduri vizuale.

\begin{figure}[h]
    \centering
    \includegraphics[width=0.4\textwidth]{app_generate_flow}
    \caption{Ecranul de Generare: Selectorii de ocazie și rezultatul vizual (Ținuta sugerată).}
    \label{fig:app_generate}
\end{figure}

Din punct de vedere al codului, componenta `OutfitCard` primește un obiect JSON de la backend și îl randează dinamic:

\begin{lstlisting}[language=JavaScript, caption={Componenta OutfitCard (simplificată)}]
export const OutfitCard = ({ outfit }) => (
  <View style={styles.card}>
    <Image source={{ uri: outfit.top_image }} style={styles.image} />
    <Image source={{ uri: outfit.bottom_image }} style={styles.image} />
    <View style={styles.info}>
      <Text style={styles.match}>Potrivire: {outfit.score}%</Text>
      <Text style={styles.reason}>{outfit.reasoning}</Text>
    </View>
  </View>
);
\end{lstlisting}

\subsection{Integrarea și Consumul API-ului}
Comunicarea cu backend-ul este centralizată în serviciul `api.ts`. Acesta gestionează automat atașarea token-ului JWT și tratarea erorilor globale (ex: expirarea sesiunii).

\begin{figure}[h]
    \centering

Această modularizare permite schimbarea ușoară a URL-ului de bază (între Development și Production) fără a modifica codul din componentele React.
